# Design Spec: Obsidian HTML Viewer Plugin (`obsidian-html-viewer`)

**Date**: 2026-06-29  
**Status**: Approved  

---

## 1. Goal & Architectural Overview

The `obsidian-html-viewer` plugin adds native rendering capabilities for `.html` files within Obsidian. It allows users to view HTML documents (such as cheat sheets, lessons, and reference materials generated by skills like `/teach`) directly inside Obsidian tabs. The plugin resolves local relative assets (CSS stylesheets, images, scripts) via Obsidian's local resource protocol (`vault.getResourcePath`), enforces security via iframe sandboxing and Content Security Policy (CSP), intercepts internal links to open `.md` notes and `.html` files in Obsidian tabs, and supports live reloading when source files change.

```

|                           Obsidian App                            |
+-------------------------------------------------------------------+
```

## 2. Module Specifications

### 2.1 `main.ts` (Plugin Entry Point)
* **Responsibilities**:
  * Registers `VIEW_TYPE_HTML` ("html-viewer-view") with Obsidian via `registerView`.
  * Registers `.html` file extension handling so clicking an HTML file opens it in `HtmlView`.
  * Adds ribbon icons and command palette entries for toggling view modes or refreshing previews.

### 2.2 `HtmlView.ts` (Workspace View Component)
* **Extends**: `ItemView`
* **State**:
  * `mode`: `'preview' | 'source'` (defaults to `'preview'`).
  * `trackedDependencies`: Set of vault file paths currently loaded by the active preview (HTML file + CSS/JS assets).
* **Key Methods**:
  * `getViewType()`: Returns `'html-viewer-view'`.
  * `getDisplayText()`: Returns the title of the HTML file.
  * `onOpen()`: Initializes containers for preview (iframe) and source editor. Adds view action buttons (toggle mode, reload) to leaf header.
  * `onLoadFile(file: TFile)`: Triggers transformation via `HtmlAssetResolver` and updates iframe `srcdoc`.
  * `handleFileModify(file: TFile)`: Debounced listener that refreshes iframe content if `file.path` matches `this.file.path` or any item in `trackedDependencies`.

### 2.3 `HtmlAssetResolver.ts` (DOM & Path Transformer)
* **Responsibilities**:
  * Parses raw HTML string using `DOMParser`.
  * Resolves relative paths (`href`, `src`, `data`, `poster`) on elements (`link`, `script`, `img`, `video`, `audio`, `source`) relative to the current file's folder in the vault.
  * Replaces relative paths with valid Obsidian local resource URLs returned by `vault.getResourcePath(targetTFile)`.
  * Injects `<meta http-equiv="Content-Security-Policy" content="...">` restricting network traffic to local schemes (`app:`, `file:`, `data:`, `blob:`).
  * Injects runtime click interceptor script into document `<head>`.
  * Returns transformed HTML string and a set of resolved asset `TFile` paths.

### 2.4 `IpcBridge.ts` (Inter-Process Communication Bridge)
* **Responsibilities**:
  * Registers global `window.addEventListener('message', ...)` handler attached to `HtmlView`.
  * Validates message origins and types (`obsidian-navigate`, `obsidian-open-external`).
  * On `obsidian-navigate`: Extracts target link text and delegates to `app.workspace.openLinkText(linkText, currentFilePath, false)`.
  * On `obsidian-open-external`: Delegates to Obsidian's external browser opening utilities.

## 3. Sandboxing, CSP & IPC Runtime Script

### 3.1 Iframe Configuration
```html
<iframe 
  sandbox="allow-scripts allow-forms allow-same-origin allow-modals" 
  style="width: 100%; height: 100%; border: none; background: var(--background-primary);"
></iframe>
```

### 3.2 Injected Runtime IPC Script
```javascript
(function() {
  document.addEventListener('click', function(e) {
    let target = e.target;
    while (target && target.tagName !== 'A') {
      target = target.parentElement;
    }
    if (!target || !target.getAttribute('href')) return;
    
    const href = target.getAttribute('href');
    if (href.startsWith('http://') || href.startsWith('https://') || href.startsWith('mailto:')) {
      e.preventDefault();
      window.parent.postMessage({ type: 'obsidian-open-external', url: href }, '*');
    } else if (!href.startsWith('#')) {
      e.preventDefault();
      window.parent.postMessage({ type: 'obsidian-navigate', linkText: href }, '*');
    }
  }, true);
})();
```

## 4. Error Handling & Edge Cases

1. **Invalid or Empty HTML**: Displays an in-pane warning card ("Empty or unparseable HTML document").
2. **Missing Vault Resources**: Missing asset references are logged as warnings and left intact without crashing the view.
3. **Lifecycle Memory Leaks**: All event handlers and message listeners are automatically registered with `this.registerEvent(...)` or explicitly cleaned up in `onClose()`.
